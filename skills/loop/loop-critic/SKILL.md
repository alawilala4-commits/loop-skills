# Loop Critic

## When to Use
Gunakan saat output perlu diuji secara kritis sebelum dipakai atau dikirim.

## Inputs
- Draft output.
- Tujuan.
- Kriteria kualitas.
- Risiko kesalahan.

## Procedure
1. Baca output secara objektif.
2. Cari celah, asumsi, dan kekurangan.
3. Bedakan error kecil dan error kritis.
4. Catat kritik dengan singkat.
5. Kirim ke verifier atau replanner.

## Output
- Daftar kritik.
- Prioritas masalah.
- Saran perbaikan.
- Status kelayakan.

## Pitfalls
- Jangan kritik tanpa dasar.
- Jangan terlalu keras sampai kehilangan solusi.
- Jangan fokus pada format saja.

## Verification
- Kritik spesifik dan berguna.
- Masalah utama terlihat jelas.
- Output siap diperbaiki.
