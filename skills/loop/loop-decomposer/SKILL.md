# Loop Decomposer

## When to Use
Gunakan saat sebuah tugas terlalu besar, kabur, atau terdiri dari banyak bagian.

## Inputs
- Tugas besar.
- Tujuan utama.
- Komponen penting.
- Constraint.
- Urutan logis.

## Procedure
1. Pahami tugas besar.
2. Identifikasi komponen utama.
3. Pisahkan menjadi subtask.
4. Hapus tumpang tindih.
5. Beri label jelas pada tiap subtask.

## Output
- Daftar subtask.
- Tujuan tiap subtask.
- Dependensi.
- Prioritas pengerjaan.
